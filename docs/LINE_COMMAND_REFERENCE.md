# LINE 指令與回覆範例

文件名稱：LINE 指令與回覆範例  
適用模式：輕量版（Sheets / LINE）  
適用對象：使用者、管理者、教育訓練人員

---

## 1. 文件目的

本文件用於說明：

- LINE 可以下哪些指令
- 系統會怎麼回覆
- 哪些屬於查詢
- 哪些屬於確認 / 核可

---

## 2. 使用前提

LINE webhook 已完成設定，且：

- 使用者帳號已在允許清單內
- `line.channel_access_token` 已設定
- `line.channel_secret` 已設定

若不在允許清單內，系統會拒絕使用。

---

## 3. 查詢類指令

### 3.1 今日待出貨

輸入：

```text
今天有哪些待出貨訂單？
```

回覆範例：

```text
✅ 今天的重點整理好了
處理對象：ORD-2026-001 / 宏達商行
系統判斷：ready_to_ship
你現在先做：先安排今日出貨，並確認包裝與出貨通知
```

### 3.2 待回覆

輸入：

```text
哪些客戶或訂單還沒回覆？
```

回覆範例：

```text
✅ 今天的重點整理好了
處理對象：ORD-2026-018 / 富利企業
系統判斷：waiting_reply
你現在先做：先回覆交期，再確認是否需要補充報價內容
```

### 3.3 缺貨與低庫存

輸入：

```text
目前哪些料件低於安全庫存？
```

回覆範例：

```text
✅ 今天的重點整理好了
處理對象：鋁板 2mm
系統判斷：shortage
目前卡住：可用庫存低於安全庫存；補貨時間可能延遲
你現在先做：先確認供應商交期，再評估是否調整出貨安排
```

### 3.4 今日新單整理

輸入：

```text
請整理今天新收到的訂單與後續動作
```

回覆範例：

```text
✅ 今天的重點整理好了
處理對象：今日新進 4 筆訂單
你現在先做：先確認其中 2 筆急單的出貨與缺料情況
```

## 4. 確認 / 核可類指令

### 4.1 查看待確認清單

輸入：

```text
approval list
```

回覆範例：

```text
📋 待確認項目
⏳ approval-3 — config:reload
⏳ approval-4 — policy:reload
```

### 4.2 查看單筆待確認細節

輸入：

```text
approval approval-3
```

回覆範例：

```text
⏳ 待確認項目 approval-3
操作：config:reload
狀態：pending
原始請求：POST /config/reload
可重試：是
你現在可以做：approve approval-3 或 reject approval-3 原因
```

### 4.3 核可

輸入：

```text
approve approval-3
```

回覆範例：

```text
✅ 已核可 approval-3
操作：config:reload
支援重試：是
你現在先做：若需重跑原始請求，請回主工作台或管理頁面繼續處理
```

### 4.4 拒絕

輸入：

```text
reject approval-4 不建議現在切換設定
```

回覆範例：

```text
🛑 已拒絕 approval-4
原因：不建議現在切換設定
```

---

## 5. 未授權回覆範例

若帳號不在允許清單內，系統會回：

```text
❌ 未經授權的使用者。請聯繫管理員將您的 LINE User ID 加入允許清單。
```

---

## 6. 未知指令回覆範例

若輸入格式錯誤，系統可能回：

```text
⚠️ 未知的核可指令。支援：approval list、approval <id>、approve <id>、reject <id> [原因]
```

---

## 7. 使用建議

### 一般使用者

建議優先使用：

- 今日待出貨查詢
- 待回覆查詢
- 缺貨查詢

### 管理者

建議另外使用：

- `approval list`
- `approval <id>`
- `approve <id>`
- `reject <id> 原因`
