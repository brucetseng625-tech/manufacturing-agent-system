## 2. 系統定位

本系統定位為：

**AI 協作分析與營運工作台**

系統角色如下：

1. 讀取企業既有資料來源
2. 針對問題進行 AI 協作分流
3. 提供風險、原因、下一步與提醒
4. 提供可視化工作場景與處理脈絡
5. 提供治理、確認、追查與稽核輔助

本系統不是 ERP 本體，不負責取代 ERP 核心交易流程。  
本系統主要提供：

- 資料理解
- 問題分析
- 任務協作
- 異常整理
- 工作建議
- 可視化監看

---

## 3. 適用模式

本系統支援以下兩種導入模式。

### 3.1 ERP 整合模式

適用情境：

- 客戶已有 ERP / MES / WMS / 內部資料庫
- 需要針對正式單號、排程、交期、商務、供應鏈資料做 AI 分析
- 需要較完整的治理、稽核、待確認、資料來源切換與追查能力

資料來源範例：

- ERP
- MES
- WMS
- 內部資料庫
- 正式主檔
- 正式單據

### 3.2 輕量營運模式

適用情境：

- 尚未導入正式 ERP
- 主要透過 LINE、Google Sheets、表單、人工整理資料
- 希望先建立輕量型日常營運整理與追蹤流程

資料來源範例：

- LINE
- Google Sheets
- 表單
- OCR / 手動輸入

## 4. 系統主要功能

### 4.1 主工作台

用途：

- 提問
- 進行工作查詢
- 依問題內容由系統分流
- 查看右側協作場景與節點詳情
- 取得原因、風險與下一步建議

### 4.2 營運治理

用途：

- 查看待確認項目
- 查看系統狀態
- 查看資料入口狀態
- 查看提醒與治理相關資訊

### 4.3 事件時間軸 / 最近動態

用途：

- 查看事件發生順序
- 追查事情從何開始、如何流轉、目前停留在哪一步

### 4.4 執行紀錄 / 處理紀錄

用途：

- 查詢某次處理結果
- 回顧某次操作、整理、查詢或回覆過程

### 4.5 技能與流程

用途：

- 說明系統目前可協助的分析與處理項目

### 4.6 使用統計

用途：

- 提供整體使用概況
- 作為導入後的管理觀察與優化參考

## 5. 畫面導覽與功能分工

系統左側主選單建議如下：

1. 主工作台
2. 營運治理
3. 事件時間軸 / 最近動態
4. 執行紀錄 / 處理紀錄
5. 技能與流程
6. 使用統計

功能分工原則：

- `主工作台`：日常工作主入口
- `營運治理`：治理與確認入口
- `事件時間軸`：脈絡追查入口
- `執行紀錄`：單筆回顧入口

---

## 6. 交付後標準操作流程

### 6.1 每日使用標準流程

1. 使用者登入後進入 `主工作台`
2. 依實際情境選擇 `ERP 整合版` 或 `輕量版`
3. 先使用「從這裡開始」入口卡
4. 若入口卡不足，再自行輸入查詢
5. 觀察右側場景哪些區域接手
6. 點選節點查看原因與下一步
7. 若有待確認事項，再進入 `營運治理`
8. 若需追查處理過程，再看 `事件時間軸` 或 `執行紀錄`

## 7. 主工作台交付說明

### 7.1 使用方式

主工作台為系統交付後的預設首頁。

使用者可透過以下方式操作：

- 點擊入口卡
- 直接輸入查詢
- 點右側協作節點

### 7.2 入口卡設計目的

入口卡用途是降低第一次使用門檻，避免使用者不知道要問什麼。

#### 輕量模式入口卡

- 先看今天要出貨什麼
- 再看誰還沒回覆
- 最後看缺貨與補救

#### ERP 整合模式入口卡

- 先查一筆訂單或問題
- 再看右邊哪一區接手
- 最後看原因與下一步

### 7.3 場景節點設計目的

右側場景用於顯示：

- 主控區
- 職能區
- 協作流向
- 目前狀態
- 異常或待確認位置

節點點擊後顯示：

- 功能說明
- 當前狀態
- 原因
- 下一步
- 相關提醒或待確認資訊

## 8. ERP 整合模式交付範圍

ERP 整合模式交付後，通常包含以下能力：

- 訂單、交期、排程、報價問題查詢
- 與正式資料來源串接

- 較完整的治理資訊

建議使用對象：

- 製造業
- 已有正式系統之企業
- 需內網部署或高資料保護需求之客戶

## 9. 輕量營運模式交付範圍

輕量模式交付後，通常包含以下能力：

- LINE 查詢入口

- 今日待出貨、待回覆、缺貨整理

## 10. LINE 與 Google Sheets 交付說明

### 10.1 LINE

系統可支援 LINE webhook 作為外部入口。

可用範圍：

- 查詢
- 指定關鍵整理項目
- 待確認項目查看
- 確認 / 拒絕類指令

交付時應確認：

- LINE channel access token
- LINE channel secret
- 允許清單設定
- 回覆格式

### 10.2 Google Sheets

系統可支援 Google Sheets 作為輕量資料來源。

可用範圍：

- 訂單整理
- 待出貨查詢
- 待回覆查詢
- 庫存或缺料查詢

交付時應確認：

- Sheet ID / gid
- 欄位命名
- CSV 匯出或讀取方式
- 輕量模式預設資料來源設定

## 12. 異常處理與支援原則

### 12.1 場景載入失敗

標準處理順序：

1. 重新整理頁面
2. 重新切換一次模式
3. 前往營運治理查看系統狀態
4. 若問題持續，通知系統管理者

### 12.2 查詢無結果

標準處理順序：

1. 檢查問題內容或單號是否正確
2. 改用入口卡查詢
3. 查看處理紀錄 / 執行紀錄
4. 若為資料問題，檢查資料來源狀態

### 12.3 待確認項目無法處理

標準處理順序：

1. 檢查使用者權限
2. 前往營運治理確認項目內容
3. 必要時由主管或管理者協助處理

### 12.4 LINE 無法回應

標準處理順序：

1. 確認帳號是否在允許清單
2. 確認 webhook 與 token 設定
3. 改由主工作台直接操作

## 13. 客戶教育訓練建議

建議交付後教育訓練分兩階段進行。

### 第一階段：一般使用者

訓練重點：

- 主工作台怎麼用
- 先按哪個入口卡
- 右側節點怎麼看
- 哪裡看原因與下一步

### 第二階段：主管 / 管理者

訓練重點：

- 營運治理怎麼看
- 事件時間軸怎麼追查
- 執行紀錄怎麼回顧
- LINE / Sheets / ERP 模式差異
